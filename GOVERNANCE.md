## Governance Model

InkScript follows a **Benevolent Dictator / Open-Control** model.

- The project is open to public contribution
- Direction is guided by community input
- Final decisions rest with the Project Owner

This model prioritizes:
- Long-term stability
- Spec consistency
- Avoidance of fragmentation

---

## Roles

### Project Owner

The Project Owner has exclusive authority to:

- Approve or reject changes to the specification
- Modify licensing terms for future releases
- Appoint maintainers or reviewers
- Resolve disputes
- Define project direction

---

### Maintainers (Optional / Appointed)

Maintainers may be appointed to:

- Review pull requests
- Triage issues
- Maintain reference implementations
- Improve documentation

Maintainers act in an advisory capacity unless explicitly delegated authority.

---

### Contributors

Contributors may:

- Submit pull requests
- Propose features
- Report issues
- Build independent tools

By contributing, contributors agree to the terms of the **SOCL 1.0**,
including the contributor submission clause.

---

## Decision-Making Process

1. Discussion occurs via issues or pull requests
2. Alternatives and trade-offs are considered
3. The Project Owner makes a final decision

Consensus is preferred but **not required**.

---
